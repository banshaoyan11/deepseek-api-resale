## 第一阶段：注册 PayPal 商家账户

### 预计时间：30 分钟

### 步骤 1.1：访问注册页面

1. 打开浏览器（推荐 Chrome）
2. 访问：**https://www.paypal.cn/portal/account-selection**
3. 选择 **"个人商家"** 或 **"企业商家"**（个人选第一个）

### 步骤 1.2：填写注册信息

```
邮箱：        建议使用 Gmail / Outlook / QQ国际版
密码：        8位，包含大小写+数字+特殊字符
手机号：      中国大陆手机号即可
身份证：      大陆身份证 + 人脸识别
```

### 步骤 1.3：完成身份认证

1. **手机验证**：收到验证码，输入即可
2. **身份证上传**：
   - 拍摄身份证正反面
   - 或使用手机拍照上传
3. **人脸识别**：按照提示完成

### 步骤 1.4：等待审核

- **审核时间**：通常 30 分钟内
- **审核结果**：会发送邮件通知

### 步骤 1.5：绑定银行卡

1. 登录 PayPal 后台
2. 进入 **"账户设置"** → **"银行账户"**
3. 添加你的中国大陆银行卡：
   ```
   银行名称：招商银行 / 工商银行 / 建设银行 等
   账户号码：你的卡号
   开户姓名：必须与身份证姓名一致
   ```
4. PayPal 会存入几分钱验证（1-2天内）

## 第二阶段：获取 API 凭证

### 预计时间：5 分钟

### 步骤 2.1：登录开发者平台

1. 访问：**https://developer.paypal.com**
2. 使用刚注册的 PayPal 账户登录
3. 首次登录会自动创建开发者账户

### 步骤 2.2：创建应用

1. 进入 **"My Apps & Credentials"**
2. 点击 **"Create App"**
3. 填写信息：
   ```
   App Name: DeepSeek API Resale (起个好记的名字)
   Environment: Sandbox (开发测试用)
   ```
4. 点击 **"Create App"**

### 步骤 2.3：复制凭证

在应用详情页面，复制以下内容：

```
Client ID:    AWxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
Secret:       xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

⚠️ **重要**：
- 开发测试用 **Sandbox** 凭证
- 正式上线用 **Live** 凭证
- 现在先用 Sandbox 测试

## 第四阶段：本地测试

### 预计时间：10 分钟

### 步骤 4.1：启动服务

```powershell
cd D:\DeepSeek_API_Resale_Platform
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

### 步骤 4.2：验证服务运行

打开浏览器访问：

| 地址 | 用途 |
|------|------|
| http://localhost:8000 | 首页 |
| http://localhost:8000/docs | API 文档 |

### 步骤 4.3：注册测试用户

使用 curl 或 Postman：

```bash
curl -X POST http://localhost:8000/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "email": "test@example.com",
    "password": "testpass123",
    "full_name": "Test User"
  }'
```

### 步骤 4.4：登录获取 Token

```bash
curl -X POST http://localhost:8000/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "test@example.com",
    "password": "testpass123"
  }'
```

响应：
```json
{
  "access_token": "eyJhbGciOiJIUzI1NiIs...",
  "token_type": "bearer"
}
```

### 步骤 4.5：创建充值订单

```bash
curl -X POST http://localhost:8000/billing/top-up \
  -H "Authorization: Bearer YOUR_TOKEN_HERE" \
  -H "Content-Type: application/json" \
  -d '{"amount": 10.00}'
```

响应：
```json
{
  "checkout_url": "https://www.sandbox.paypal.com/checkoutnow?token=XXX",
  "order_id": "XXX",
  "provider": "paypal"
}
```

### 步骤 4.6：完成 PayPal 支付

1. 复制 `checkout_url` 到浏览器
2. 使用 **Sandbox 测试账户**登录：
   - 访问：https://developer.paypal.com/developer/accounts
   - 找到 **Personal Account**
   - 登录信息：
     ```
     Email: sb-xxxxx@person.com
     Password: xxxxxxxx
     ```
3. 完成支付

### 步骤 4.7：捕获支付并验证余额

```bash
curl -X POST http://localhost:8000/billing/paypal/capture/YOUR_ORDER_ID
```

### 步骤 4.8：验证余额增加

```bash
curl -X GET http://localhost:8000/billing/balance \
  -H "Authorization: Bearer YOUR_TOKEN_HERE"
```

应该看到余额从 0 变成 10.00。

## 第五阶段：部署到 Railway

### 预计时间：15 分钟

### 步骤 5.1：上传代码到 GitHub

```powershell
cd D:\DeepSeek_API_Resale_Platform

# 初始化 Git（如果还没初始化）
git init

# 添加所有文件
git add .

# 提交
git commit -m "Initial commit - DeepSeek API Resale Platform"

# 添加远程仓库（替换 YOUR_USERNAME）
git remote add origin https://github.com/YOUR_USERNAME/deepseek-api-resale.git

# 推送
git branch -M main
git push -u origin main
```

### 步骤 5.2：创建 Railway 项目

1. 访问：**https://railway.app**
2. 登录（可用 GitHub 账户）
3. 点击 **"New Project"**
4. 选择 **"Deploy from GitHub repo"**
5. 选择你的仓库

### 步骤 5.3：配置环境变量

在 Railway 控制台，进入 **"Variables"** 添加：

```env
# 数据库（Railway 会自动提供）
DATABASE_URL=postgresql://xxx

# DeepSeek
DEEPSEEK_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxx
DEEPSEEK_BASE_URL=https://api.deepseek.com

# PayPal（Live 凭证）
PAYPAL_CLIENT_ID=AWxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
PAYPAL_CLIENT_SECRET=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
PAYPAL_SANDBOX=false

# 支付提供商
PAYMENT_PROVIDER=paypal

# JWT
SECRET_KEY=生成一个随机字符串

# 基础URL（替换为你的域名）
BASE_URL=https://yourdomain.com

# 定价
API_PRICING_INPUT=0.30
API_PRICING_OUTPUT=0.50
```

### 步骤 5.4：配置域名（可选）

1. 在 Railway 项目设置中添加自定义域名
2. 在你的域名提供商添加 CNAME 记录
3. 等待 DNS 生效

### 步骤 5.5：部署

Railway 会自动检测 Python 项目并部署。

## ❓ 常见问题

### Q1: 支付后余额没到账？

**可能原因**：
1. Webhook 配置错误
2. 网络问题
3. 订单 ID 不匹配

**解决方法**：
1. 检查 Railway 日志
2. 手动调用 capture 接口
3. 验证 PayPal 交易记录

### Q2: 如何处理退款？

**流程**：
1. 登录 PayPal 后台
2. 找到交易记录
3. 发起退款
4. 手动调整用户余额

### Q3: 手续费是多少？

| 交易类型 | PayPal 费率 |
|---------|------------|
| 国内交易 | 0.5% |
| 国际交易 | 1.5% |

### Q4: 资金多久到账？

| 银行 | 到账时间 |
|------|---------|
| 工商银行 | 即时 |
| 其他银行 | 1-3 工作日 |
